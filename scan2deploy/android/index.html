Scan2Deploy Android

Scan2Deploy is an application devoted to the initial staging of Android devices by reading a barcode sequence. Depending on the content of the staging barcodes Scan2Deploy will, in this order:

  • configure the device Wi-Fi network,
  • control some device inner system settings (e.g. the lock-screen enabled state),
  • download a data archive from a HTTP/HTTPS server and unpack it to the device,
  • install/update applications and the system image, and
  • run a script (to perform any additional setup).
note

The documentation on this page is current as of Scan2Deploy Android version v1.27.

Downloads#

Downloads are available in the Releases section on the Scan2Deploy Android Github page.

Enable Scan2Deploy#

Scan2Deploy acts as an Android Enterprise Device Policy Controller (DPC), running in device owner mode. This allows Scan2Deploy to configure many protected device settings that would otherwise not be possible. Any DPC app that acts as a device owner must be configured to do so during device setup. By scanning a QR code at the first page of the Android Setup Wizard ("Hi There! page) on a new or factory-reset device, Scan2Deploy is configured as device owner.

note

The simplest way to enable Scan2Deploy is to just scan any Scan2Deploy QR code at the Hi There! page.

Alternatively, an Android Enterprise QR code can be scanned.

Android Enterprise QR code for pre-installed Scan2Deploy#

Scan the following QR code at the Hi There! page to enroll the version of Scan2Deploy pre-installed in the device firmware as device owner:

note

Android Enterprise QR Code Generator#

The Android Enterprise QR Code Generator can be used to install the latest version on Scan2Deploy as well as give you the opportunity to add many other customization options. The basic process is:

  1. Select Datalogic Scan2Deploy in the Select an EMM field.
  2. Make any additional settings as desired (for instance, configure a Wi-Fi network).
  3. Press the Generate QR Code button at the bottom of the page.
  4. Scan the QR code displayed.

Creating Scan2Deploy files#

Scan2Deploy JSON files can be created either by Scan2Deploy Studio or manually using a text editor. It's highly recommended to use Scan2Deploy Studio.

Scan2Deploy Studio#

Refer to the Scan2Deploy Studio documentation for a detailed description of how to use the tool.

Using a text editor#

Scan2Deploy JSON files can be created using any text editor (notepad.exe can be used for example). A JSON schema is available to help ensure valid file content. There are several advantages to using an editor that supports this schema:

  • Provides help text for each field
  • Provides real-time compiler-like messages letting you know when you have made a mistake
  • Allows you to write files faster and with fewer mistakes

$schema#

A JSON schema file is used to define the structure and features available in Scan2Deploy. The $schema tag is used to indicate which version of the schema you are using. You can versions of schema along with complete field documentation at the Scan2Deploy Android schema repo.

ðŸ”— Schema Reference

The latest schema can be referenced like this:

{
"$schema": "https://raw.githubusercontent.com/datalogic/scan2deploy-android-schema/master/schema.json"
}

You can also point to a specific version of the schema by pointing to the desired tag. For example:

{
"$schema": "https://raw.githubusercontent.com/datalogic/scan2deploy-android-schema/v1-1-0/schema.json"
}

Choosing an editor#

There are several good JSON editors available. We recommend Visual Studio Code. It is free and has many features designed to make writing JSON files easier, including utilizing JSON schemas.

By including the aforementioned $schema tag in you JSON file, Visual Studio Code will provide automatic tool tips and error underlining:

Some other editors know to use the schema files on schemastore.org when a given file uses a specific file extension that is registered on schemastore.org: file-name.dla.json

Third-party integration#

Scan2Deploy configuration data can be passed to Scan2Deploy agent directly from other Android applications. For instance an MDM agent installed on the device could directly pass barcode data to Scan2Deploy agent, eliminating the need for a barcode to be physically scanned. The following steps are necessary to achieve this.

Generate Scan2Deploy data#

This can be achieved either by using dl_config.exe or by using Scan2Deploy Studio.

Using dl_config.exe#

Once you have created a valid Scan2Deploy JSON file, use the --encodeonly flag of dl_config.exe (v1.7 or later) to cause the encoded data to be displayed on screen. Example usage:

dl_config.exe test.json X --encodeonly=true
Datalogic Staging Configurator (ver 1.7.1524)
Copyright (c) 2016-2017 by Datalogic MCSA S.r.l.
======================================================================
Loading file 'test.json'
Processing data
-----BEGIN ENCODED DATA-----
Bm9uiYc//NWOnA0ZtIrgJ/qy0DCvHR+a1XwUz1sTne8s0KEM9ANwTS0XD4NbIOf/hV0+MM7cOFiZMbUDtO1L4sb4vnJluFbQOEM3phIuCRiBuMtpnh85
-----END ENCODED DATA-------

From there, copy the data found between the -----BEGIN ENCODED DATA----- and -----END ENCODED DATA----- headers and proceed to the startActivity section below.

Using Scan2Deploy Studio#

Once you have stepped through Scan2Deploy Studio and have your barcode displayed on the Save and Print page, click the Copy to Clipboard button to copy the data to your copy buffer. The Copy to Clipboard button is available in Scan2Deploy Studio 1.3 and later.

Start Service intent#

Scan2Deploy offers a service that listens for requests to apply Scan2Deploy profiles without the Scan2Deploy UI (activity) being visible. The profile will be applied silently, without any Scan2Deploy UI being displayed in the foreground.

  • Broadcast action: datalogic.scan2deploy.intent.action.START_SERVICE
  • Broadcast receiver: com.datalogic.scan2deploy.S2dServiceReceiver

The calling app can get notified when Scan2Deploy finishes processing the data by providing package and receiver string extras in the intent. If these extras are present, Scan2Deploy will broadcast the feedback intent to specified receiver at the end of execution.

Extras#

ExtraTypeRequiredDefaultDescription
packagestringnonullPackage name of the calling app (example: com.android.test). If provided, receiver should be provided also.
receiverstringnonullBroadcast receiver of the calling app to receive the feedback intent (example: com.android.test.FeedbackReceiver). If provided, package should be provided also.
encodingenumnov2Indicates how the data field is encoded. Allowed values are:
â€¢ json - plain, unencrypted JSON file

Examples#

These examples demonstrate passing properly escaped json data in data extra. The encoding extra must be set to json in this case.

  • Java
  • Shell
Intent i = new Intent("datalogic.scan2deploy.intent.action.START_SERVICE")
.putExtra("package", "com.android.test")
.putExtra("receiver", "com.android.test.FeedbackReceiver")
.putExtra("encoding", "json")
.putExtra("data", "{\"global\":{\"action\":\"reset\"}}");
.setClassName("com.datalogic.scan2deploy", "com.datalogic.scan2deploy.S2dServiceReceiver");
getApplicationContext().sendBroadcast(i);
adb shell am broadcast -a datalogic.scan2deploy.intent.action.START_SERVICE -n "com.datalogic.scan2deploy/.S2dServiceReceiver" --es encoding v2 --es data "{\"global\":{\"action\":\"reset\"}}" --es package "com.android.test" --es receiver "com.android.test.FeedbackReceiver"

Feedback intent#

Below are the details of the feedback intent sent back to the original application if requested (by providing package and receiver extras in the calling intent).

  • Broadcast intent: com.datalogic.scan2deploy.FEEDBACK
info

Prior to Scan2Deploy agent 1.29, the broadcast intent was: com.datalogic.scan2depoly.feedback. Notice the scan2depoly misspelling and the fact that FEEDBACK was lowercase.

ExtraTypeAlways present?Description
feedbackstringyesJson string with two keys, status, and info. status will contain either success or error. info is a freeform message describing what occurred. It will generally contain all the output of that would normally be displayed in the Scan2Deploy UI output window

Example feedback receiver:

@Override
public void onReceive(Context context, Intent intent) {
Log.i(TAG, "onReceive");
if (intent.getAction().equals("com.datalogic.scan2deploy.FEEDBACK")) {
try {
JSONObject feedbackJson = new JSONObject(intent.getStringExtra("feedback"));
if(feedbackJson.getString("status").equalsIgnoreCase("success")) {
Log.d(TAG, "Success");
} else if(feedbackJson.getString("status").equalsIgnoreCase("error")) {
Log.d(TAG, "Error");
}
}catch (JSONException err){
Log.d(TAG, err.toString());
}
}
}

Call startActivity#

Alternatively, you can start the Scan2Deploy activity if you want the Scan2Deploy UI to be visible. The same intent extras can be used in this case (encoding and data). These examples demonstrate passing in v2 barcode data exported by Scan2Deploy Studio in the data extra. Since v2 is the default, no encoding parameter is required.

  • Java
  • Shell
Intent launchIntent = getPackageManager().getLaunchIntentForPackage("com.datalogic.scan2deploy");
launchIntent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK);
launchIntent.putExtra("data", "Bm9uiYc//NWOnA0ZtIrgJ/qy0DCvHR+a1XwUz1sTne8s0KEM9ANwTS0XD4NbIOf/hV0+MM7cOFiZMbUDtO1L4sb4vnJluFbQOEM3phIuCRiBuMtpnh85");
getApplicationContext().startActivity(launchIntent);
adb shell am start -S -n com.datalogic.scan2deploy/.MainActivity -e data Bm9uiYc//NWOnA0ZtIrgJ/qy0DCvHR+a1XwUz1sTne8s0KEM9ANwTS0XD4NbIOf/hV0+MM7cOFiZMbUDtO1L4sb4vnJluFbQOEM3phIuCRiBuMtpnh85
note

You can utilize an MDM console to start the Scan2Deploy activity or service with the desired data intent extra. However, that process will differ significantly from one MDM to the next and is outside the scope of what can be documented here.

Logging#

There are 2 ways to capture log details after scanning Scan2Deploy barcodes on the device; In app and Using ADB. The In app method is usually easier and doesn't require enabling special features on the device, while the adb method does require enabling special features but provides more detailed information to aid in debugging.

In app#

  • Click the 3 dots icon on the upper-right hand side of the screen.

  • Click Share...

  • Select the method you would like to use to share the output. For example, Copy to clipboard.

Using adb#

  • Enable ADB on your device. This can be accomplished by turning on the Enable USB debugging setting under System Settings > USB on the Device Settings page in Scan2Deploy Studio. Alternatively, follow Google's directions on your device.

  • Connect your device to your PC over USB.

  • While at a command prompt that has access to the adb command, type:

    adb logcat -d Scan2Deploy:V *:S > s2d.log

  • Now, share the s2d.log file as needed.
