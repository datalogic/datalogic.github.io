Browsing

Overview​

The browser screen allows the user to display content from web applications. It is shown once a site from the Favorites screen is tapped.

Navigation bar​

At the top of the browser screen is a navigation bar with information and controls for assisting with web page navigation. When full screen page viewing is enabled (the default), it can be exposed by dragging the page down. It can be hidden again by dragging up.

The contents of the navigation bar is configurable. Additional details on the options are found in Display Settings. By default, the navigation bar is configured to display the title of the current web page. This can be changed to display the URL of the current page:

When displayed, the URL cannot be edited. You can only browse to web pages using hyperlinks or an item on the Favorites screen. If both the title and URL display are disabled, that portion of the navigation bar is hidden. The remaining buttons are for navigating to web pages and browser screens.

ButtonDescription
HomeBrowses to the Home page. If no page has been specified in the Browser configuration, this will display the Favorites Screen.
BackBrowses to the previous web page in the navigation history.
ForwardBrowses to the next web page in the navigation history.
RefreshReloads the current web page.
FavoritesOpens the Favorites screen.
SettingsOpens the Administrator password dialog, before going to the Browser Settings screen if the correct password is entered.
Note: This option may not appear if it has been disabled.

Browsing pages​

Normally, browsing to web pages continues without issue by tapping on hyperlinks or by tapping on an icon from the Favorites screen. The following sections describe special behavior when browsing to pages.

Error pages​

When browsing web pages, the Datalogic Enterprise Browser will block any attempt to load a page which is located in a domain not in the allowed list. When this happens, an error page will load which displays the bad URL and the reason for the failure.

In addition, various (device or server) network errors can occur, or a page may timeout trying to load. When this happens, an error page is also displayed which display the nature of the error and an error code to assist in diagnosing the root cause.

Unsafe browsing​

Some websites require additional certification in order to form a secure connection. If the browser detects an error it can recover from, it may display a dialog such as the following:

The dialog will contain a message briefly describing the nature of the error. It is generally unsafe to continue browsing to such a website, but the user it given the option to continue anyway. Once a selection is made, it will be remembered for the duration of the browsing session.

Client certificates​

Some websites require a client certificate to be installed on the device in order to browse to its pages. If detected, the browser will display a dialog listing one or more potential matching certificates for the user to choose from before proceeding to the site. Once the user has selected a certificate, the browser will proceed to the web page (or display an error if the certificate is not a match). The browser will remember the certificate for the duration of the browsing session.

Page authentication​

Some web pages require authentication before the page loads. In the case of basic authentication (requiring a username and password), the browser will present a dialog identifying the web server and requesting authentication credentials.

Once the credentials have been entered, tap Submit to send the information to the server for verification. If the information is not correct, the dialog will appear again to retry. This will continue to happen until the credentials are accepted. Once accepted, they are stored for future use the next time the browser visits the same web server. These credentials can also be specified in the Site Configuration before visiting the site.
