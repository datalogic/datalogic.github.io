Scan2Deploy Android 1.27

November 3, 2020 · 1 min read

Scan2Deploy Android

Enhancements#

  • Support the SET_SETTING script command.

  • Supports applying profiles located on device in internal storage, on removable USB storage, and on removable SD cards. See storage-type in the deployment section of the Android schema.

  • Allows plain json to be passed to Scan2Deploy via the START_SERVICE intent and when starting the Scan2Deploy activity.

Compatibility#

  • This version of Scan2Deploy supports all schema versions up to and including schema v1-9-0. Schema v1-9-0 is required for the SET_SETTINGS script command and the storage-type field.

  • This version is designed to work with Scan2Deploy Studio 1.4.3 or higher. Note that version 1.4.3 does not yet offer direct support for the new SET_SETTINGS script command and storage-type fields.

Corrected defects#

  • Resolves an issue that prevented some MDMs from successfully passing barcode data to the Scan2Deploy service.
