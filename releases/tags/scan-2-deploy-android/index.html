Scan2Deploy Android 1.27

November 3, 2020 · 1 min read

Scan2Deploy Android

Enhancements#

  • Support the SET_SETTING script command.

  • Supports applying profiles located on device in internal storage, on removable USB storage, and on removable SD cards. See storage-type in the deployment section of the Android schema.

  • Allows plain json to be passed to Scan2Deploy via the START_SERVICE intent and when starting the Scan2Deploy activity.

Scan2Deploy Android 1.26

October 6, 2020 · 1 min read

Scan2Deploy Android

Compatibility#

  • This version of Scan2Deploy supports the following schemas: schema v1-0-0, schema v1-1-0, schema v1-2-0, schema v1-3-0, schema v1-4-0, schema v1-5-0, schema v1-6-0, schema v1-7-0.
  • This version is designed to work with Scan2Deploy Studio 1.4.3 or higher.

Corrected defects#

  • Enabling multi-scan can make Scan2Deploy agent unable to read new barcodes
  • Deployment doesn't resume after performing a self-update to Scan2Deploy agent
  • Scan2Deploy agent can exit on certain devices when the scanned profile includes disable of Manage Apps (using the DISABLE_APP script command)

Scan2Deploy Android 1.25

September 22, 2020 · 1 min read

Scan2Deploy Android

Enhancements#

  • Added support for downloading profiles using FTP. See scheme, ftp-username, and ftp-password in the deployment section of the Android schema.
  • Added WPA3 support for some devices running Android 9 and above. See new mode values in the deployment section of the Android schema.
  • Allow setting a different system default launcher. See new default-home section in the Android schema.
  • Added a service that listens for requests to apply Scan2Deploy profiles without the Scan2Deploy UI (activity) being visible. The profile will be applied silently, without any Scan2Deploy UI being displayed in the foreground.
